wamm Master Activation

1) Checkout

1) Satın Alma

Connect your payment provider checkout URL in docs/site-config.json under buyUrl. Use Stripe Checkout, Lemon Squeezy, Paddle, or your provider of choice.

Ödeme sağlayıcınızın checkout adresini docs/site-config.json içindeki buyUrl alanına bağlayın. Stripe Checkout, Lemon Squeezy, Paddle veya tercih ettiğiniz sağlayıcı kullanılabilir.

2) License Issue

2) Lisans Üretimi

Generate a signed license token (JWT or signed blob) after payment. Include plan, expiry, and seat/device policy in claims.

Ödeme sonrası imzalı lisans tokenı (JWT veya imzalı blob) üretin. Plan, süre ve cihaz/koltuk politikalarını token claim’lerine ekleyin.

3) App Activation

3) Uygulama Aktivasyonu

In desktop app settings, add an activation panel that accepts the token and verifies signature locally. Optional online check can validate revocation, but audio must never be uploaded.

Masaüstü ayarlara lisans token alan aktivasyon paneli ekleyin ve imzayı yerelde doğrulayın. İsteğe bağlı online kontrol yalnızca iptal bilgisini doğrulayabilir; ses dosyası asla yüklenmemelidir.

4) Waitlist CTA

4) Bekleme Listesi CTA

Configure waitlistUrl in docs/site-config.json. Current suggested flow uses GitHub Discussions or an external form.

docs/site-config.json içinde waitlistUrl alanını ayarlayın. Önerilen akış GitHub Discussions veya harici form bağlantısıdır.

5) Satış öncesi zorunlu yasal paket

  • Privacy (Local-first)
  • Third-party notices
  • Model license notices
  • EULA

  • Üçüncü taraf bildirimleri
  • Model lisans bildirimleri
